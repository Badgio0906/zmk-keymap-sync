# 利用仕様（前提条件）

> 注: このファイルは既存の要件定義書 `docs/requirements.md` とは別に、
> エンドユーザー向けの利用前提条件を明記するために新設した。

## 前提条件：対応ファームウェアが必須

本アプリ（ZMK キーマップ同期アプリ）は、以下の条件を満たす
**ZMK Studio 対応ファームウェア**が書き込まれたキーボードでのみ動作する。

- ファームウェアが `CONFIG_ZMK_STUDIO=y` でビルドされていること
- `studio-rpc-usb-uart` スニペットが適用され、RPC通信が USB 経由で
  行えるようになっていること（本アプリは WebSerial 経由で接続するため）

この2条件のどちらかが欠けている場合、実機への接続（USB接続ボタン）・
実機からの逆読み込み・実機への即時書き込みのいずれも動作しない。

## 対応ビルドかどうかの確認方法

対象キーボードのファームウェア設定リポジトリ（例:
`zmk-config-roBa`）の `build.yaml` を開き、該当シールドのビルド
エントリに以下が揃っているかを確認する。

```yaml
- board: xxx
  shield: <対象シールド名>
  snippet: studio-rpc-usb-uart   # ← これが無いと実機には繋がらない
```

加えて、シールドの `.conf` ファイル（例:
`config/boards/shields/<shield>/<shield>.conf`）に
`CONFIG_ZMK_STUDIO=y` が設定されていることも確認する。
